# RoCathon: Semantic Projection Engine — Final Submission
**Participant: Varun Sharma**

## 🎯 Executive Summary
This repository contains a fully compliant, locally reproducible hybrid search engine designed to solve the exact business problem mathematically: surfacing creators who possess both the perfect aesthetic context ("vibe") and a proven track record of commercial ROI.

This solution avoids the pitfalls of "pure semantic AI search" by rigorously enforcing commerce filters against the AI layer. 

---

## 🏗 System Architecture & Implementation Highlights

### 1. Vector Database Setup
*   **Database Engine:** PostgreSQL hosted on Supabase.
*   **Vector Extension:** Enabled `pgvector`.
*   **Custom Table Schema:** The `creators` table was provisioned dynamically via script, utilizing a `vector(384)` column engineered specifically to match the output dimensionality of the chosen embedding model.

### 2. The AI Layer (Local Embeddings Strategy)
Rather than relying on paid external APIs (like OpenAI) which can break during judging or require complex credential passing, this solution utilizes `@xenova/transformers`.
*   **Model Used:** `Xenova/all-MiniLM-L6-v2`.
*   **Why?** It runs 100% locally and completely isolated inside the Node.js process. This guarantees that any judge reviewing this code can run `npm run ingest` immediately without bringing their own API keys.
*   **Strategic Context Indexing:** The embeddings are generated by combining the creator's `bio` and their `content_style_tags` into a single, dense semantic context string.

### 3. The Retrieval Pipeline (Strict Adherence to Constraints)
The core logic resides in `src/searchCreators.ts` and processes a query like *"Affordable home decor for small apartments"*.

1.  **Semantic Targeting (Top 50 Filter):** The raw query is embedded locally and passed to Postgres. An intensive explicit constraint was added: `LIMIT 50`. The database utilizes the `1 - (embedding <=> $1)` Cosine Similarity function to return strictly the Top 50 creators based *purely* on aesthetics before any business biases are introduced.
2.  **Hybrid Combination Formula:**
    *   `Final score = (Semantic x A) + (Projected x B)`
    *   **Baseline Weights:** Started with `0.45` for Semantic and `0.55` for Projected.

### 4. The "Zero Tradeoff" Algorithm (Commerce Reality Check)
The hardest constraint of the hackathon was: *"High vibe / zero GMV must rank lower than good vibe / high GMV"*.
*   **Implementation:** Inside the `computeHybridScore()` function, there is a dynamic weight penalizer: `if (metrics.total_gmv_30d === 0)`.
*   **The Result:** If an artist has generated zero gross merchandise volume, their multiplier coefficients are algorithmically shattered down to `0.1`. This ensures that they literally cannot outrank a creator who possesses merely "acceptable" semantic similarities but produces massive real-world ROI.

## 🚀 Step-by-Step Instructions (For Judges to Reproduce)

This system was designed for zero-friction reproduction.

### Step 1: Install Dependencies
Ensure you have a modern version of Node.js installed on your machine.
```bash
git clone <this-repo>
cd roc-hackathon
npm install
```

### Step 2: Environment Setup
Locate the `.env` file in the root directory. If none exists, copy the `.env.example`:
```bash
cp .env.example .env
```
Ensure your `DATABASE_URL` is configured to point to your Supabase/Postgres instance:
```env
# Example
DATABASE_URL=postgresql://postgres:[password]@db...supabase.co:5432/postgres
# NOTE: No OpenAI API keys are required to execute this application because embeddings are handled 100% locally via Xenova.
```

### Step 3: Database Ingestion & Schema Creation
The ingestion script does all the heavy lifting. It connects to Postgres, creates the `vector` extension, drops any existing `creators` table, recreates the strict schema, embeds the `creators.json` data, and populates the database.
```bash
npm run ingest
```
*(Wait approximately 30-45 seconds for all local embeddings to generate and upload).*

### Step 4: Run the Hybrid Search Engine
To execute the required test case (query: *Affordable home decor...* against the `brand_smart_home` profile):
```bash
npm run demo
```
**Output:** The script queries the database, applies the Zero Tradeoff hybrid weights, cuts the list to the Top 10 Ranked Creators, and automatically drops a `submission_top10.json` file securely into the root directory. 

✅ **Final Status:** All automated scripts execute with exit code 0. All mandatory fields and schemas strictly enforced.
